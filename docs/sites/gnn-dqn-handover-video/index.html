Training flow

Each episode creates experience from moving UEs, then replay trains the model

Training uses scenario mixing, epsilon exploration, target networks, optional prioritized replay, n-step returns, and held-out SON validation.

1. Pick scenariodense urban, highway, Pokhara, event, rural, holdout validation.
2. Advance mobilityUEs move, shadowing updates, coherent RSRP/RSRQ snapshot refreshes.
3. Act and storestate, action, reward, next state, done, valid mask.

Replay mini-batch

B graphs are copied into one disconnected batch: B times N cell nodes. The code passes batch_size and nodes_per_graph so a 40-cell topology is never mistaken for two 20-cell graphs.

Deployable algorithm

SON-GNN-DQN turns learned preferences into safe network knobs

Raw GNN-DQN is a research baseline. The defense-ready method is the SON layer that adjusts bounded CIO and TTT while standard A3 still executes the handover.

CIO bounded: -6 dB to +6 dB
TTT adapts when ping-pong rises

Preference share

Count how many UEs served by cell A prefer target B.

Rollback

Undo CIO changes if throughput drops or ping-pong increases.

RSRP_target + CIO(source,target) > RSRP_serving + A3_offset condition must hold for TTT steps GNN preference becomes a safe parameter update

Deployment path

From drive-test compatible UE_ONLY to future O-RAN/E2

O-RAN is future/demo, not the main headline
1. UE reportsRSRP, RSRQ, trend, speed, serving state, handover history.
2. GNN-DQNScores per-UE target preferences over the topology graph.
3. SON layerAggregates preferences and updates bounded CIO/TTT values.
4. Standard A3Executes handover only when offset and TTT conditions are satisfied.
5. O-RAN/E2Later adds true PRB and KPM counters through near-RT RIC integration.

Drive-test data without PRB remains valid: PRB utilization is zero and PRB availability is zero. RSRQ-derived load is always described as a proxy, never as real PRB utilization.
